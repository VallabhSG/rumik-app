1. System Overview

Rumik is a React Native (Expo) mobile application augmented with a custom Over-The-Air (OTA) update control plane. The system enables staged JS bundle rollouts, real-time feature flag management, crash-triggered auto-rollback, performance monitoring, and a full CI/CD pipeline across seven GitHub Actions workflows.

The architecture is split into three tiers: the mobile client, the OTA control plane server, and the CI/CD pipeline. An admin single-page application provides operational visibility across all three tiers.

Mobile Client

  • React Native + Expo
  • expo-updates (bundle delivery)
  • TypeScript
  • AsyncStorage
  • WebSocket client

OTA Server

  • Node.js + Express
  • TypeScript
  • better-sqlite3
  • Zod validation
  • WebSocket (ws)

CI/CD & Infra

  • GitHub Actions (7 workflows)
  • EAS Build (Expo)
  • Vercel (web)
  • Firebase (Android)
  • TestFlight (iOS)

5. OTA Update Distribution Flow

JavaScript bundles are delivered via Expo's CDN. The OTA control plane manages eligibility, rollout percentage, and crash-triggered rollback independently.

App->>App: nativeVersion in [min, max] range? alt Eligible for update App->>CP: POST /api/update-events { event_type: eligible } App->>Expo: checkForUpdateAsync() Expo-->>App: Update manifest available App->>CP: POST /api/update-events { event_type: download_start } App->>Expo: fetchUpdateAsync() App->>CP: POST /api/perf-metrics { metric: update_download_ms } App->>CP: POST /api/update-events { event_type: applied } App->>App: reloadAsync() on next foreground else Not eligible App->>CP: POST /api/update-events { event_type: skipped } end Admin->>CP: GET /api/update-events/funnel?release_id=...
